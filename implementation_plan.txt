# Next.js + Vercel Migration Plan
## From WordPress/GitHub Pages → Vercel (empowerandlink.com)

---

## Step 1: `next.config.ts` — Remove static export config

Replace the GitHub Pages static export config with Vercel-native config.

**Changes:**
- Remove `output: 'export'`
- Remove `trailingSlash: true`
- Remove `basePath` and `assetPrefix`
- Remove `env.NEXT_PUBLIC_BASE_PATH`
- Keep `images: { unoptimized: true }` (images are static files)

**Before:**
```ts
const isProd = process.env.NODE_ENV === 'production';
const basePath = isProd ? '/empowerandlink' : '';

const nextConfig: NextConfig = {
  output: 'export',
  trailingSlash: true,
  images: { unoptimized: true },
  basePath,
  assetPrefix: basePath,
  env: {
    NEXT_PUBLIC_BASE_PATH: basePath
  }
};
```

**After:**
```ts
const nextConfig: NextConfig = {
  images: { unoptimized: true }
};
```

## Step 2: `app/utils/paths.ts` — Simplify `getAssetPath`

On Vercel at root domain, no prefix is needed. `getAssetPath` returns the path as-is.

```ts
export const getAssetPath = (path: string): string => {
  const normalizedPath = path.startsWith('/') ? path : `/${path}`;
  return normalizedPath;
};
```

(Or remove the function and inline bare paths — but keeping the function minimizes diffs.)

No callers need to change since the function signature is identical; only the return value changes.

## Step 7: Vercel Setup (manual — web UI)

1. Go to https://vercel.com and import the GitHub repo
2. Framework preset: **Next.js**
3. Build command: `npm run vercel-build`
4. Output directory: `.next` (default)
5. Add environment variables:
   - `NEXT_PUBLIC_ADMIN_PASSWORD` — the admin password
   - `NEXT_PUBLIC_GITHUB_TOKEN` — GitHub fine-grained PAT
6. Deploy

---

## Step 8: Custom Domain (manual — requires DNS access)

1. In Vercel dashboard → Project → Settings → Domains
2. Add `empowerandlink.com`
3. Vercel provides DNS targets (e.g., `cname.vercel-dns.com`)
4. Update DNS records at your domain registrar / DNS provider:
   - Replace current WordPress A/CNAME records with Vercel's target
5. Wait for DNS propagation (5-30 min)
6. Vercel provisions SSL certificate automatically

## Notes

- The admin page at `/admin/` uses a GitHub PAT committed client-side. This is acceptable for a fine-grained PAT with `contents:write` scoped to the single repo.
- `NEXT_PUBLIC_GITHUB_TOKEN` and `NEXT_PUBLIC_ADMIN_PASSWORD` are compile-time environment variables. They must be set in Vercel env vars before deploy.
- If build fails with ESLint warnings-as-errors, add `eslint: { ignoreDuringBuilds: true }` to `next.config.ts` as a quick fix, or fix the lint issues.
